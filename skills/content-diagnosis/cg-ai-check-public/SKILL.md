# AI 写作特征检查

## 核心判断

AI 味的问题通常不是“写得差”，而是写得太稳、太满、太顺。

重点检查：

- 过度解释
- 空泛连接词
- 套路化排比
- 讲义腔
- 工作流语言
- 过度正确但没有信息量
- 段落过于均匀
- 结尾强行升华

## 工作原则

1. 默认先诊断，不直接重写。
2. 标出具体句子，不只给抽象评价。
3. 区分“AI 味”和“选题本身有问题”。
4. 修改建议要更像真人表达，不是换一套高级词。

## 检查清单

### 1. 开头

- 是否用“在当今时代”“随着 XX 发展”这类空泛开场。
- 是否先解释背景，而不是先进入问题。
- 是否第一句话就能让读者知道发生了什么。

### 2. 中段

- 是否每段都太完整、太平均。
- 是否大量使用“首先、其次、最后、总之”。
- 是否把读者已经知道的事又解释一遍。
- 是否有“不是 X，而是 Y”的过度套路。

### 3. 结尾

- 是否强行拔高。
- 是否用抽象词收束。
- 是否给了正确但无行动价值的话。

### 4. 真实感

- 有没有具体场景。
- 有没有作者自己观察到的细节。
- 有没有不完美但真实的表达。

## 输出结构

```markdown
## AI Check

**总体判断**
- AI 味等级：低 / 中 / 高
- 最大问题：

**命中句子**
1. 原句：
   问题：
   修改方向：

2. 原句：
   问题：
   修改方向：

**优先修改顺序**
1. ...
2. ...
3. ...

**是否建议重写**
- 是 / 否
- 原因：
```

## 边界

- 如果选题本身不成立，先建议做内容诊断。
- 如果只是错别字和语法，不要夸大成 AI 味。
- 不处理法律、医疗、投资等专业事实判断。

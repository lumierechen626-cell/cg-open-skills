# 知识库处理

## 定位

这只 Skill 不是做逐字稿，而是把原始资料处理成可检索、可复用、可讲解的结构化知识。

## 适用输入

- PDF
- 截图
- 录音转写
- 培训资料
- 产品资料
- FAQ
- Markdown 文稿

## 默认目标

- 判断资料类型。
- 提取关键信息。
- 分清事实、解释、限制和动作。
- 输出后续可学习、讲解、比对的结构化文档。

## Step 1：资料识别卡

```markdown
【资料识别卡】
- 资料名称：
- 资料类型：
- 来源形式：
- 当前清晰度：
- 最终想怎么用：
```

## Step 2：选择输出模式

默认四种：

- 条款 / 规则解读
- 方案整理
- 课程知识卡
- FAQ

用户没指定时，先判断哪种最有用。

## Step 3：信息分层

至少分成三层：

- 原始事实
- 解释性归纳
- 仍不确定的地方

如果 OCR 不清、截图残缺、录音漏字，要标出来。

## 输出结构

```markdown
# 结构化知识卡

## 资料识别
...

## 核心结论
...

## 关键信息
- ...

## 限制 / 边界
- ...

## 常见误解
- ...

## 可复用表达
- ...

## 后续动作
- ...
```

## 规则

- 不把推断写成事实。
- 不补原资料没有的内容。
- 不保留用户私有路径。
- 不输出无法回溯来源的结论。

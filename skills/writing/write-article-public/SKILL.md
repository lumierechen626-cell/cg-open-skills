# 长文写作

## 定位

这只 Skill 处理单篇文章写作，不负责完整公众号发布链路。

如果用户要从选题到发布全流程，应使用长文链条。

## 工作流程

### 1. 确认文章任务

先确认：

- 写给谁看。
- 文章想解决什么问题。
- 文章类型：观点、教程、复盘、行业分析、案例拆解。
- 已有材料有哪些。
- 是否需要调研。

### 2. 建主线

先写：

- 核心判断
- 反命题
- 读者为什么要关心
- 需要证明什么

### 3. 写大纲

大纲必须说明每一节的作用，不只列标题。

### 4. 写初稿

初稿先求完整，不追求漂亮。

### 5. 精修

分两轮：

- 内容轮：逻辑、事实、证据。
- 表达轮：节奏、可读性、开头、结尾。

## 输出结构

```markdown
# 文章写作包

## 文章任务
- 目标读者：
- 核心判断：
- 文章类型：

## 大纲
1. ...
2. ...

## 初稿
...

## 精修建议
- 内容问题：
- 表达问题：
- 需要补的证据：
```

## 规则

- 不假设用户有内部知识库。
- 没有证据的判断要标出来。
- 不把摘要写成压缩版正文。
- 不把长文写成讲义或说明书。

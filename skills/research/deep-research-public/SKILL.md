# 深度调研

## 定位

这只 Skill 负责把一个模糊主题变成可验证的证据池和调研报告。

## 适用场景

- 行业研究
- 政策变化
- 竞品分析
- 产品调研
- 内容写作前调研
- 观点文章证据补充

## 调研类型

先判断本次属于哪一类：

- 事实核查：回答“这件事是不是真的”。
- 行业扫描：回答“这个领域现在发生了什么”。
- 竞品分析：回答“别人怎么做、做到什么程度”。
- 决策支持：回答“我接下来要不要做、怎么做”。
- 写作证据：为文章补数据、案例和反证。

## 工作流程

### 1. 明确调研问题

先把主题改写成可查的问题：

- 要回答什么。
- 不回答什么。
- 时间范围。
- 地区范围。
- 需要哪些来源。

输出一张子问题卡：

```markdown
【调研子问题卡】
- 主问题：
- 必须回答：
- 可以不回答：
- 时间范围：
- 地区范围：
- 需要核查的关键事实：
```

### 2. 建证据池

至少分四类：

- 官方来源
- 行业报告
- 媒体报道
- 一线案例 / 用户声音

来源优先级：

1. 官方文件、监管公告、企业公告、原始数据。
2. 学术论文、行业协会、权威报告。
3. 主流媒体和专业媒体。
4. 企业官网、产品页面、用户评论。
5. 社交平台内容只能作为线索，不能单独作为结论。

### 3. 标注可靠性

每条信息标注：

- 来源
- 时间
- 可信度
- 是否需要二次核验

如不同来源互相冲突，必须单独列出：

- 冲突点是什么。
- 哪个来源更接近一手信息。
- 目前能不能下结论。

### 4. 输出报告

```markdown
# 调研报告

## 调研问题
...

## 结论摘要
...

## 关键发现
1. ...
2. ...
3. ...

## 证据池
| 信息 | 来源 | 时间 | 可信度 | 备注 |
|---|---|---|---|---|

## 仍不确定
- ...

## 对后续行动的影响
- ...
```

## 规则

- 最新事实必须核查。
- 不把单一来源当结论。
- 不为了观点好看而隐藏反证。
- 高风险判断必须标注不确定性。
- 报告末尾必须保留来源列表，方便用户回查。

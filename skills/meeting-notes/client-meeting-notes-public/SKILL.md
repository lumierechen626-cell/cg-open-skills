# 客户面谈纪要

## 适用场景

- 首次需求访谈
- 方案讲解
- 回访记录
- 咨询沟通
- 服务复盘

## 输入

用户可提供：

- 会议原文
- 聊天记录
- 通话转写
- 现场记录

如果目的不清，先问：

- 这份纪要给谁看：自己 / 团队 / 客户。
- 这次沟通目标是什么。
- 会后最重要的下一步是什么。

## 处理流程

### 1. 提取事实

优先提取：

- 客户基本情况
- 当前关注点
- 已确认需求
- 主要顾虑
- 已给出的建议
- 会议结论
- 下一步动作

### 2. 区分事实和推断

必须分开写：

- 客户明确说过什么。
- 你从上下文推断了什么。

### 3. 判断客户状态

- 刚建立联系
- 正在识别需求
- 正在比较方案
- 卡在异议
- 接近成交
- 成交后服务

判断状态时必须写一句理由，不能只贴标签。

## 顾虑与异议区分

- 顾虑：客户还没有想清楚，重点是补信息、降理解成本。
- 异议：客户已经有明确反对点，重点是判断反对点来自价格、信任、需求、时机还是替代方案。

不要把所有犹豫都当成异议，也不要把客户沉默解读成有成交意向。

## 成交后 / 服务复盘

如果这次沟通发生在成交后，下一步动作优先写：

- 服务确认
- 材料补齐
- 风险提醒
- 回访时间
- 转介绍是否适合提，不适合就不要硬提

## 输出格式

```markdown
# 客户面谈纪要

## 本次沟通摘要
...

## 客户画像
- 身份 / 场景：
- 当前阶段：
- 最关心的问题：

## 明确需求
- ...

## 主要顾虑
- ...

## 已给出的说明或方案
- ...

## 会议结论
- ...

## 下一步动作
- 负责人：
- 动作：
- 截止时间：

## 跟进提醒
- 下次联系建议：
- 建议发送材料：
- 不要再说的话：

## 可直接发送的跟进消息
...
```

## 质量标准

- 以客户原意为准，不美化。
- 不补不存在的成交意向。
- 结论能回溯到原始记录。
- 纪要能指导下一次触达。
